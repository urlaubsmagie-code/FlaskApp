2. Add inbox-specific functions:

`createConversationCard(conv)` - Creates new conversation card HTML:
- Returns an `<a>` element with same structure as Jinja template
- Platform icon based on conv.platform (email/whatsapp/airbnb/else)
- Guest name: conv.guest.name || conv.guest.email || 'Unknown Guest'
- Time: format timestamp (use formatRelativeTime from app.js if needed, or simple date formatting)
- Subject, preview (first 80 chars of last_message.content)
- Platform badge, AI badge (if ai_enabled), status badge
- Set data-conversation-id and data-updated-at

`updateConversationCard(card, conv)` - Updates existing card in place:
- Update time span
- Update preview text
- Update status badge
- Update data-updated-at attribute
- Do NOT replace entire card (prevents flicker)

`updateInboxList(conversations)` - Main update function:
- Get container element (#conversationList)
- Build map of existing cards by ID
- For each conversation in response:
  - If card exists with same ID:
    - Compare data-updated-at with conv.updated_at
    - If different, call updateConversationCard()
    - Move to correct position if needed (newest first)
  - Else (new conversation):
    - Create card with createConversationCard()
    - Insert at correct position
- Remove empty state div if conversations exist
- Handle empty case: show empty state if no conversations

3. Initialize PollingManager:
```javascript
const inboxPoller = new PollingManager({
    fetchFn: async (signal) => {
        const response = await fetch('/chatbot/api/conversations?per_page=50', { signal });
        if (!response.ok) throw new Error(`HTTP ${response.status}`);
        return response.json();
    },
    onUpdate: (data) => {
        updateInboxList(data.conversations);
    },
    interval: 15000  // 15 seconds
});

// Start polling when page loads
document.addEventListener('DOMContentLoaded', () => {
    inboxPoller.start();
});
```

4. Update the existing `refreshConversations()` function to trigger a manual poll instead of page reload:
```javascript
function refreshConversations() {
    inboxPoller.stop();
    inboxPoller.start(); // This triggers immediate poll
}
```

- Match the exact HTML structure from the Jinja template for new cards
- Preserve filter state when updating (respect current display:none on filtered cards)
