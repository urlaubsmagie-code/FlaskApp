  <files>static/js/polling.js</files>
  <action>
Create `static/js/polling.js` with an ES6 PollingManager class:

Constructor options:
- `fetchFn`: Async function that receives AbortController signal, returns data
- `onUpdate`: Callback invoked with data when fetch succeeds
- `interval`: Polling interval in ms (default 15000)
- `onError`: Optional error callback (default logs to console)

Instance properties:
- `timeoutId`: Current setTimeout ID (null when not scheduled)
- `abortController`: Current AbortController (null when no request in flight)
- `isPolling`: Boolean state flag

Methods:
- `start()`: Begin polling (idempotent - no-op if already polling)
  - Sets isPolling = true
  - Adds visibilitychange event listener
  - Calls _poll() immediately (no initial delay)

- `stop()`: Stop polling completely
  - Sets isPolling = false
  - Removes visibilitychange listener
  - Calls _cancelPending()

- `_poll()`: Internal - execute one poll cycle
  - Guard: return early if !isPolling or document.visibilityState === 'hidden'
  - Abort any existing request (abortController.abort())
  - Create new AbortController
  - Call fetchFn(signal)
  - On success: if still isPolling, call onUpdate(data) then _scheduleNext()
  - On error: if not AbortError and still isPolling, call onError then _scheduleNext()

- `_scheduleNext()`: Internal - schedule next poll
  - Call _cancelPending() first (clear any existing timeout)
  - Set timeoutId = setTimeout(() => this._poll(), this.interval)

- `_cancelPending()`: Internal - cancel pending timeout and request
  - clearTimeout(timeoutId), set to null
  - abortController.abort(), set to null

- `_onVisibilityChange()`: Internal - visibility change handler (bound in constructor)
  - If document.visibilityState === 'visible': call _poll() immediately
  - Else: call _cancelPending()

Key patterns:
- Use recursive setTimeout (NOT setInterval) to prevent call stacking
- Always create NEW AbortController for each request (they can only abort once)
- Bind _onVisibilityChange in constructor for proper `this` context
- Guard every async callback with isPolling check

Add JSDoc comments for public methods.
