  <name>Task 1: Create FilterState module with URL sync</name>
  <files>static/js/filter-state.js</files>
  <action>
Create `static/js/filter-state.js` with the following FilterState class:

**State structure:**
```javascript
{
    platform: null,  // null = all, or 'email'|'whatsapp'|'airbnb'|'booking'
    status: null     // null = all, or 'active'|'pending_owner'|'closed'
}
```

**Constructor:**
- Call `loadFromURL()` to initialize state from URL params
- Add `popstate` event listener that calls `loadFromURL()`, `applyFilters()`, `updateUI()`

**Methods:**
1. `loadFromURL()` - Read `platform` and `status` from `URLSearchParams(window.location.search)`
2. `saveToURL()` - Use `history.replaceState()` to update URL without creating history entry:
   - Set param if value is truthy, delete if null
   - Use `new URL(window.location.href)` to build URL properly
3. `setPlatform(platform)` - Set platform state, call saveToURL + applyFilters + updateUI
4. `setStatus(status)` - Set status state, call saveToURL + applyFilters + updateUI
5. `reset()` - Reset both filters to null, call saveToURL + applyFilters + updateUI
6. `clearPlatform()` - Shorthand for setPlatform(null)
7. `clearStatus()` - Shorthand for setStatus(null)
8. `applyFilters()` - Apply visibility to `.conversation-card` elements:
   - Card visible if `matchesPlatform AND matchesStatus AND matchesSearch`
   - matchesPlatform: `!this.state.platform || card.dataset.platform === this.state.platform`
   - matchesStatus: `!this.state.status || card.dataset.status === this.state.status`
   - matchesSearch: Check search input value against card.textContent (lowercase match)
   - Set `card.style.display = (all match) ? 'flex' : 'none'`
9. `updateUI()` - Update filter button active states and active filter indicators:
   - Toggle `.active` class on `[data-filter-platform]` buttons matching current platform
   - Toggle `.active` class on `[data-filter-status]` buttons matching current status
   - Call `updateFilterIndicators()`
10. `updateFilterIndicators()` - Render active filter badges to `#activeFilters` container:
    - Clear container innerHTML
    - If platform set, append platform badge with close button
    - If status set, append status badge with close button
    - Show/hide `#clearFiltersBtn` based on whether any filter is active
11. `createFilterBadge(type, value)` - Create badge element:
    - Class: `active-filter-badge {type}-{value}`
    - innerHTML: formatted value + close button with onclick calling `clearPlatform()` or `clearStatus()`
    - Use `escapeHtml()` for value display (function must be available from inbox.html)
12. `formatFilterValue(type, value)` - Format for display:
    - Platform: capitalize first letter (email -> Email)
    - Status: replace underscores with spaces, title case (pending_owner -> Pending Owner)

**Export:**
```javascript
// Create singleton and expose globally
const filterState = new FilterState();
// Don't auto-apply on construction - let inbox.html call when ready
```

**Key pattern from research:** Use `history.replaceState` not `pushState` to avoid cluttering browser history.
