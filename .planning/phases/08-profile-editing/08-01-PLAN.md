  <name>Task 1: Add PATCH endpoint for guest basic info</name>
  <files>routes.py</files>
  <action>
Add a new route `api_update_guest` at `/api/guests/<int:guest_id>` with method `PATCH`.

Implementation details:
1. Get guest by ID using `Guest.query.get_or_404(guest_id)`
2. Parse JSON body with `request.get_json()`
3. Return 400 if no data provided
4. If `email` is in data and differs from current:
   - Check for existing guest with that email: `Guest.query.filter_by(email=data['email']).first()`
   - If exists and is different guest, return 409 with `{'error': 'Email already in use'}`
5. Update allowed fields (name, email, phone):
   - Trim whitespace: `data['name'].strip() if data['name'] else None`
   - Only update if key present in data (partial update support)
6. Commit and return `guest.to_dict()`

Place this route AFTER the existing `api_get_guest` route (line ~276) to keep guest routes grouped.

Follow existing code style:
- Use docstring: `"""Update guest basic info (name, email, phone)"""`
- Use same error response pattern as other routes
  </action>
  <verify>
Manual curl test:
```bash

  -H "Content-Type: application/json" \

```

  <action>
Add a new route `api_update_guest_detail` at `/api/guests/<int:guest_id>/details/<int:detail_id>` with method `PATCH`.

Implementation details:
1. Get detail using `GuestDetail.query.filter_by(id=detail_id, guest_id=guest_id).first_or_404()`
   - This ensures the detail belongs to the specified guest
2. Parse JSON body with `request.get_json()`
3. Validate `detail_value` is present: return 400 with `{'error': 'detail_value is required'}` if missing
4. Validate non-empty after trim: return 400 with `{'error': 'detail_value cannot be empty'}` if whitespace-only
5. Update the detail:
   - `detail.detail_value = new_value`
   - `detail.confidence = 1.0` (manual edits have full confidence)
6. Commit and return `detail.to_dict()`

Place this route AFTER the existing `api_delete_guest_detail` route (line ~308) to keep detail routes grouped.

Follow existing code style with docstring: `"""Update a guest detail value"""`
